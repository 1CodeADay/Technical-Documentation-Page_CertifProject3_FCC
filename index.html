Overview

The "Simple To-Do List API" is designed to help users manage their daily tasks. It allows for the creation, retrieval, updating, and deletion of tasks through a series of HTTP requests. This API is structured around RESTful principles, meaning that it uses standard HTTP methods (GET, POST, PUT, DELETE) to perform operations on resources identified by URLs.

This guide will walk you through each endpoint available in the API, provide examples of request and response formats, and explain how to set up and test the API using Postman.

Base URL: http://localhost:3000/api/v1

The Base URL is the root address where the API is hosted. All endpoints will be appended to this base URL to form the full address used in each API call.

Endpoints

Each endpoint in this API is designed to handle specific operations related to managing tasks. Below, we describe each endpoint, the HTTP method used, the purpose of the endpoint, the required inputs, and the expected outputs.

  1. Get All Tasks
    • HTTP Method: GET
    • Endpoint: /tasks
    • Description: This endpoint retrieves a complete list of all tasks in the to-do list. It's useful for displaying all tasks to the user at once, providing an overview of what needs to be done and what has already been completed.
    • Expected Response:
      • The response will be a JSON array, where each element represents a task, including its unique ID, title, and completion status (true for completed, false for incomplete).
    • Status Code: 200 OK
      • This status code indicates that the request was successful, and the server is returning the requested data.

    Example of how the JSON response might look:

    [ { "id": 1, "title": "Buy groceries", "completed": false }, { "id": 2, "title": "Walk the dog", "completed": true } ]

    This endpoint is typically used when you want to display all tasks to a user, for example, in a to-do list application where the user sees all their tasks listed on the main screen.

  2. Get a Single Task
    • Endpoint: /tasks/:id
    • Method: GET
    • DescriptionFetches the details of a specific task identified by its ID. This is particularly useful when a user wants to view or edit the details of a specific task.
    • Parameters: id-- The unique identifier of the task you want to retrieve. This is included in the URL.
    • Expected Response:
      • The server responds with a JSON object representing the task, including its ID, title, and completion status.

      • Example of how the JSON response might look:

      { "id": 1, "title": "Buy groceries", "completed": false }

      This endpoint is commonly used when a user selects a task from a list to view more details or edit the task.

  3. Create a New Task
    • Endpoint: /tasks
    • Method: POST
    • DescriptionAdds a new task to the to-do list. The user provides the task's title, and the server assigns a unique ID and sets the task as incomplete by default.
    • Request Body:
      • The body of the request should be a JSON object containing the task title. For example:
        • { "title": "Clean the house" }
    • Expected Response: The server responds with a JSON object representing the newly created task, including its ID, title, and completion status.
    • Status Code:
      • 201 Created: The task was successfully created and added to the list.

    Example of how the JSON response might look:

    { "id": 3, "title": "Clean the house", "completed": false }

    This endpoint is used when a user adds a new task, for example, when they click an "Add Task" button in a to-do list application.

  4. Update a Task
    • Endpoint: /tasks/:id
    • Method: PUT
    • DescriptionUpdates the details of a specific task identified by its ID. The user provides the new title and completion status for the task.
    • Parameters: id-- The unique identifier of the task you want to update. This is included in the URL.
    • Request Body:
      • The body of the request should be a JSON object containing the updated task details. For example:
        • { "title": "Clean the house", "completed": true }
    • Expected Response:
      • The server responds with a JSON object representing the updated task, including its ID, title, and completion status.
    • Status Code: 200 OK
      • The task was successfully updated.

    Example of how the JSON response might look:

    { "id": 3, "title": "Clean the house", "completed": true }

    This endpoint is used when a user edits the details of an existing task, for example, when they change the title or mark a task as completed.

  5. Delete a Task
    • Endpoint: /tasks/:id
    • Method: DELETE
    • DescriptionRemoves a specific task from the to-do list. The task is identified by its unique ID.
    • Parameters: id-- The unique identifier of the task you want to delete. This is included in the URL.
    • Expected Response:
      • The server responds with a JSON object representing the deleted task, including its ID, title, and completion status.
    • Status Code: 204 No Content
      • The task was successfully deleted.

    Example of how the JSON response might look:

    { "id": 3, "title": "Clean the house", "completed": true }

    This endpoint is used when a user removes a task from the list, for example, when they click a "Delete" button next to a task in a to-do list application.

Using Postman

Postman is a popular tool for testing APIs and making HTTP requests. It provides a user-friendly interface for sending requests, viewing responses, and organizing your API testing workflow. Here's how you can set up and test the "Simple To-Do List API" using Postman:

  1. Download and Install Postman

    If you don't already have Postman installed, you can download it from the official website: https://www.postman.com/downloads/. Follow the installation instructions for your operating system.

  2. Open Postman

    Once Postman is installed, open the application on your computer. You should see the Postman interface with options to create a new request, import collections, and more.

  3. Create a New Request

    Click on the "New" button in the top-left corner of the Postman window to create a new request. You can choose to create a request in a new tab or a new window.

  4. Enter the Request URL

    In the request tab, enter the full URL of the endpoint you want to test. For example, if you want to retrieve all tasks, the URL would be: http://localhost:3000/api/v1/tasks.

  5. Choose the HTTP Method

    Use the dropdown menu next to the URL to select the appropriate HTTP method for the request. For example, if you are retrieving data, you would use the GET method.

  6. Send the Request

    Click the "Send" button to send the request to the API. Postman will display the response from the server, including the status code, headers, and body of the response.

  7. View the Response

    You can view the response body in various formats, such as JSON, XML, or HTML, depending on the content type

Examples

Here are some examples of how to use the "Simple To-Do List API" endpoints:

  1. Get All Tasks

    2. To retrieve all tasks, send a GET request to the following URL:

    http://localhost:3000/api/v1/tasks

    This will return a JSON array containing all the tasks.

  2. Get a Single Task

    3. To retrieve a specific task, send a GET request to the following URL, replacing :id with the actual ID of the task:

    http://localhost:3000/api/v1/tasks/:id

    This will return a JSON object representing the task with the specified ID.

  3. Create a New Task

    4. To create a new task, send a POST request to the following URL:

    http://localhost:3000/api/v1/tasks

    The request body should contain a JSON object with the task details, such as the title. The server will assign a unique ID to the task and set it as incomplete by default.

  4. Update a Task

    5. To update a specific task, send a PUT request to the following URL, replacing :id with the actual ID of the task:

    http://localhost:3000/api/v1/tasks/:id

    The request body should contain a JSON object with the updated task details, such as the new title and completion status.

  5. Delete a Task

    6. To delete a specific task, send a DELETE request to the following URL, replacing :id with the actual ID of the task:

    http://localhost:3000/api/v1/tasks/:id

    This will remove the task from the to-do list.
